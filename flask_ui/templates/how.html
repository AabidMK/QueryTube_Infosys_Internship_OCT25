How It Works

QueryTube uses semantic search powered by vector embeddings to find relevant video segments across our dataset.

Overview

  • Embedding: Each video transcript is converted into a fixed-size vector using a sentence-transformer model.
  • Vector DB: Vectors are stored in ChromaDB for fast nearest-neighbor search.
  • Search: Your query is embedded and compared to stored vectors to return the most semantically similar videos.
  • Similarity Score: We compute a similarity percentage to represent relevance.
  • Summaries: You can view an AI-generated summary for each video.

Privacy & Data

Uploaded CSVs are used to augment your local dataset. No data is sent externally unless you configure an external API.
